# 01 补充：Flutter 是什么

对应主文档章节：`1. Flutter 是什么`

## 先建立一个整体印象

Flutter 不是单纯的组件库，而是一整套跨平台 UI 开发方案。

- 用 Dart 编写
- 用 Widget 描述界面
- 一套代码可以构建多端应用

## 你真正要学的不是“组件名字”

初学者最容易误以为 Flutter 就是背组件。

其实更重要的是：

- 页面怎么由 Widget 组合出来
- 数据变化后界面为什么会刷新
- 页面之间怎么跳转
- 布局为什么会报错或溢出

## 学 Flutter 的正确预期

- 先能看懂基础页面
- 再能自己拼出简单页面
- 最后再进入状态管理、网络请求、工程化结构

## 初学者先抓住哪几件事

如果你刚开始学 Flutter，最值得先抓住的是：

- Widget 是界面的基本单位
- 布局决定组件怎么摆
- 状态决定界面为什么会变
- 路由决定页面怎么切换

先把这四件事建立直觉，后面学起来会顺很多。

## Flutter 和传统前端的直观区别

可以先粗略理解成：

- HTML/CSS/JS 更像“结构 + 样式 + 逻辑”分开
- Flutter 更像“用 Dart 直接把界面树描述出来”

所以你会经常看到：

- 组件套组件
- 参数里再传组件
- 参数里再传函数

这也是 Flutter 代码看起来层层嵌套的原因。
