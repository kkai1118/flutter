# 02 补充：开发环境和项目结构

对应主文档章节：`2. 开发环境和项目结构`

## 入门阶段最需要认识的文件

- `lib/main.dart`：应用入口
- `lib/`：业务代码主要都写在这里
- `pubspec.yaml`：依赖、资源、版本配置
- `assets/`：图片、字体等静态资源
- `test/`：测试代码

## 初学者先完成三件事

1. 能用 `flutter run` 把项目跑起来
2. 知道页面代码通常写在 `lib/` 下
3. 知道图片资源要先在 `pubspec.yaml` 里声明

## 先别在工程结构上纠结太久

入门时知道“入口、页面、资源、依赖”这四个位置就够用了。

## 一个最小阅读顺序

拿到一个 Flutter 项目后，可以先按这个顺序看：

1. `lib/main.dart`
2. `MaterialApp` 在哪里
3. 首页是谁
4. 页面文件放在哪
5. 资源和依赖在哪里配

这样比一上来翻很多目录更不容易迷路。

## `pubspec.yaml` 最常见的两个作用

### 1. 配依赖

比如：

```yaml
dependencies:
  flutter:
    sdk: flutter
```

### 2. 配资源

比如：

```yaml
flutter:
  assets:
    - assets/images/
```

初学阶段先记住：图片放进项目里还不够，通常还要在这里声明。
